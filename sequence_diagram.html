2.9. Xây dựng lược đồ Tuần tự (Sequence Diagram)

Phân lớp kiến trúc MVC - Mô hình BCE (Boundary - Control - Entity) chuẩn UML

2.9.1. Lược đồ Tuần tự Đăng ký (Signup)

Quy trình đăng ký tài khoản bắt đầu khi người dùng yêu cầu chức năng trên giao diện biên Form Đăng ký. Biểu mẫu được hiển thị để người dùng nhập thông tin cần thiết. Hệ thống điều khiển tiếp nhận thông tin và truy vấn xuống cơ sở dữ liệu (lớp thực thể CSDL) để xác minh tài khoản đã tồn tại hay chưa. Nếu thông tin hợp lệ, hệ thống tiến hành gửi mã OTP xác thực qua email khách hàng. Khách hàng nhập đúng mã OTP, hệ thống sẽ thực hiện ghi nhận bản ghi tài khoản mới vào cơ sở dữ liệu và thông báo đăng ký thành công. Trường hợp thông tin không hợp lệ hoặc người dùng chủ động hủy, quy trình sẽ kết thúc và hiển thị thông báo thất bại.

⏳ Đang tự động vẽ sơ đồ từ PlantUML...
@startuml
skinparam defaultFontName Arial
skinparam sequence {
  ActorBackgroundColor #E0D7FF
  ActorBorderColor #000000
  BoundaryBackgroundColor #FFFDE0
  BoundaryBorderColor #000000
  ControlBackgroundColor #D1FAE5
  ControlBorderColor #000000
  EntityBackgroundColor #FEE2E2
  EntityBorderColor #000000
  LifeLineBorderColor #333333
  LifeLineBackgroundColor #BFDBFE
}

actor User as user
boundary "Form đăng ký" as boundary
control "Xử lý thông tin" as control
entity "Dữ liệu" as entity

user -> boundary : Chọn chức năng đăng ký
activate boundary
boundary -> user : Hiển thị form đăng ký
deactivate boundary

user -> boundary : Nhập thông tin tài khoản
activate boundary
boundary -> control : Gửi yêu cầu kiểm tra()
activate control
control -> entity : Kiểm tra tài khoản tồn tại()
activate entity
entity -> control : Trả kết quả (Chưa tồn tại)
deactivate entity

control -> user : Gửi mã OTP xác nhận về Email
deactivate control
deactivate boundary

user -> boundary : Nhập mã OTP & xác nhận
activate boundary
boundary -> control : Yêu cầu xác thực OTP()
activate control
control -> entity : Thêm tài khoản mới vào DB()
activate entity
entity -> control : Xác nhận thêm thành công
deactivate entity
control -> boundary : Thông báo thành công
deactivate control
boundary -> user : Hiển thị thông báo tạo tài khoản thành công
deactivate boundary

alt #FEE2E2 Trường hợp thông tin không hợp lệ hoặc hủy thao tác
  user -> boundary : Nhấn Hủy đăng ký / Nhập thông tin sai
  activate boundary
  boundary -> control : Hủy bỏ / Gửi dữ liệu lỗi()
  activate control
  control -> boundary : Trả về thông báo lỗi (Thông tin không hợp lệ)
  deactivate control
  boundary -> user : Hiển thị thông báo đăng ký thất bại
  deactivate boundary
end
@enduml

2.9.2. Lược đồ Tuần tự Đăng nhập (Login)

Người dùng truy cập vào Biểu mẫu Đăng nhập và cung cấp email cùng mật khẩu định danh. Hệ thống tiếp nhận thông tin và gửi yêu cầu xác thực sang bộ xử lý điều khiển. Bộ xử lý thực hiện truy xuất dữ liệu từ bảng người dùng thông qua Email để nhận về chuỗi mật khẩu đã được băm (hash) và salt. Hệ thống tự động tiến hành so khớp mã băm. Nếu trùng khớp, hệ thống khởi tạo Token/Session làm việc, thông báo thành công và chuyển hướng người dùng về trang chủ. Ngược lại, nếu sai thông tin, hệ thống trả về thông báo lỗi chi tiết trên biểu mẫu hiển thị.

⏳ Đang tự động vẽ sơ đồ từ PlantUML...
@startuml
skinparam defaultFontName Arial
skinparam sequence {
  ActorBackgroundColor #E0D7FF
  ActorBorderColor #000000
  BoundaryBackgroundColor #FFFDE0
  BoundaryBorderColor #000000
  ControlBackgroundColor #D1FAE5
  ControlBorderColor #000000
  EntityBackgroundColor #FEE2E2
  EntityBorderColor #000000
  LifeLineBorderColor #333333
  LifeLineBackgroundColor #BFDBFE
}

actor User as user
boundary "Form đăng nhập" as boundary
control "Bộ xử lý đăng nhập" as control
entity "CSDL" as entity

user -> boundary : Chọn chức năng đăng nhập
activate boundary
boundary -> user : Hiển thị form đăng nhập
deactivate boundary

user -> boundary : Nhập Email & Mật khẩu
activate boundary
boundary -> control : Gửi yêu cầu xác thực()
activate control
control -> entity : Truy xuất tài khoản theo Email()
activate entity
entity -> control : Trả về mật khẩu hash & salt()
deactivate entity
control -> control : So khớp mật khẩu hash()
alt #D1FAE5 Mật khẩu chính xác
  control -> control : Khởi tạo Session / Token()
  control -> boundary : Xác thực thành công
  boundary -> user : Chuyển hướng về Trang chủ & Lưu Session
else #FEE2E2 Mật khẩu sai hoặc không tồn tại tài khoản
  control -> boundary : Trả về thông báo lỗi (Sai thông tin)
  deactivate control
  boundary -> user : Hiển thị thông báo sai tài khoản/mật khẩu
  deactivate boundary
end
@enduml

2.9.3. Lược đồ Tuần tự Đặt lịch hẹn sửa chữa (Booking)

Khách hàng chọn chức năng đặt lịch trên giao diện và biểu mẫu thông tin lịch hẹn được hiển thị. Khách hàng thực hiện cung cấp thông tin mô tả lỗi máy cùng khung giờ mong muốn. Hệ thống đặt lịch tiếp nhận dữ liệu và thực hiện truy xuất cơ sở dữ liệu để đối chiếu lịch làm việc và khả năng tiếp nhận của kỹ thuật viên trong khung giờ đó. Nếu khung giờ trống khả dụng, hệ thống lập tức thêm mới bản ghi lịch hẹn vào CSDL và gửi thông báo thành công. Trường hợp khung giờ đã vượt quá công suất tiếp nhận, hệ thống từ chối và yêu cầu khách hàng lựa chọn khung giờ khác thích hợp.

⏳ Đang tự động vẽ sơ đồ từ PlantUML...
@startuml
skinparam defaultFontName Arial
skinparam sequence {
  ActorBackgroundColor #E0D7FF
  ActorBorderColor #000000
  BoundaryBackgroundColor #FFFDE0
  BoundaryBorderColor #000000
  ControlBackgroundColor #D1FAE5
  ControlBorderColor #000000
  EntityBackgroundColor #FEE2E2
  EntityBorderColor #000000
  LifeLineBorderColor #333333
  LifeLineBackgroundColor #BFDBFE
}

actor "Người dùng" as user
boundary "Trang Đặt lịch" as boundary
control "Hệ thống Đặt lịch" as control
entity "CSDL" as entity

user -> boundary : Chọn Đặt lịch hẹn
activate boundary
boundary -> user : Hiển thị Form nhập lịch hẹn
deactivate boundary

user -> boundary : Nhập hãng máy, lỗi & lựa chọn thời gian
activate boundary
boundary -> control : Gửi yêu cầu kiểm tra lịch trống()
activate control
control -> entity : Truy xuất lịch làm việc & KTV()
activate entity
entity -> control : Trả về số lượng đặt lịch hiện tại()
deactivate entity
alt #D1FAE5 Khung giờ còn trống
  control -> entity : Tạo lịch hẹn mới()
  activate entity
  entity -> control : Lưu lịch hẹn thành công
  deactivate entity
  control -> boundary : Xác nhận thành công
  boundary -> user : Hiển thị thông báo Đặt lịch thành công
else #FEE2E2 Khung giờ đã đầy / Không hợp lệ
  control -> boundary : Thông báo (Khung giờ đã đầy)
  deactivate control
  boundary -> user : Hiển thị cảnh báo & yêu cầu chọn thời gian khác
  deactivate boundary
end
@enduml
Hình 27. Lược đồ Tuần tự - Đặt lịch hẹn sửa chữa
2.9.4. Lược đồ Tuần tự Mua sắm & Thanh toán (Checkout)

Khi khách hàng tiến hành thanh toán giỏ hàng, hệ thống biên hiển thị trang thanh toán và tự động gọi bộ xử lý để kiểm tra và áp dụng mã giảm giá (voucher) nếu có. Người dùng nhập địa chỉ nhận hàng, chọn phương thức thanh toán (COD hoặc trực tuyến) và xác nhận. Bộ điều khiển thực hiện gọi dữ liệu kho hàng để đối chiếu số lượng tồn kho khả dụng của sản phẩm trong giỏ. Nếu sản phẩm đủ hàng, hệ thống xử lý cổng thanh toán, tạo bản ghi đơn hàng mới, cập nhật trừ số lượng hàng tồn kho tương ứng trong CSDL và gửi thông báo checkout thành công. Trường hợp phát hiện sản phẩm đã hết hàng trong quá trình xử lý, hệ thống trả về thông báo lỗi hết hàng.

⏳ Đang tự động vẽ sơ đồ từ PlantUML...
@startuml
skinparam defaultFontName Arial
skinparam sequence {
  ActorBackgroundColor #E0D7FF
  ActorBorderColor #000000
  BoundaryBackgroundColor #FFFDE0
  BoundaryBorderColor #000000
  ControlBackgroundColor #D1FAE5
  ControlBorderColor #000000
  EntityBackgroundColor #FEE2E2
  EntityBorderColor #000000
  LifeLineBorderColor #333333
  LifeLineBackgroundColor #BFDBFE
}

actor "Khách hàng" as user
boundary "Trang Thanh toán" as boundary
control "Hệ thống Thanh toán" as control
entity "CSDL" as entity

user -> boundary : Nhấn Thanh toán trong Giỏ hàng
activate boundary
boundary -> control : Yêu cầu tính tổng tiền & Voucher()
activate control
control -> entity : Kiểm tra tính hợp lệ của Voucher()
activate entity
entity -> control : Trả về thông tin Voucher()
deactivate entity
control -> boundary : Hiển thị tổng số tiền đã giảm trừ
deactivate control
deactivate boundary

user -> boundary : Nhập địa chỉ giao hàng & Xác nhận thanh toán
activate boundary
boundary -> control : Yêu cầu đặt hàng()
activate control
control -> entity : Kiểm tra số lượng tồn kho sản phẩm()
activate entity
entity -> control : Trả về số lượng tồn kho()
deactivate entity
alt #D1FAE5 Đủ tồn kho
  control -> control : Xử lý thanh toán trực tuyến / COD()
  control -> entity : Lưu đơn hàng mới & Trừ số lượng tồn kho()
  activate entity
  entity -> control : Xác nhận cập nhật CSDL thành công
  deactivate entity
  control -> boundary : Đặt hàng thành công
  boundary -> user : Chuyển hướng đến trang thông báo thành công
else #FEE2E2 Vượt quá tồn kho khả dụng
  control -> boundary : Trả về lỗi (Sản phẩm hết hàng)
  deactivate control
  boundary -> user : Hiển thị thông báo sản phẩm trong kho không đủ
  deactivate boundary
end
@enduml

2.9.5. Lược đồ Tuần tự Quản lý Sản phẩm & Kho hàng (Admin)

Hành động của Quản trị viên bắt đầu bằng cách đăng nhập và truy cập trang quản lý sản phẩm để xem danh sách. Khi Admin thực hiện khởi tạo sản phẩm mới, hệ thống điều khiển tiếp nhận dữ liệu sản phẩm, kiểm tra tính hợp lệ của thông tin và ghi nhận vào cơ sở dữ liệu với số lượng tồn kho khởi tạo bằng 0. Tiếp theo, Admin thực hiện nhập kho hàng hóa bằng cách lập phiếu đơn nhập kho cho sản phẩm đó. Hệ thống điều khiển xác thực thông tin phiếu nhập, sau đó lưu bản ghi đơn nhập kho đồng thời tự động cộng dồn số lượng tồn kho thực tế của linh kiện trong cơ sở dữ liệu, cuối cùng hiển thị kết quả số lượng tồn kho mới lên màn hình của Admin.

  ActorBorderColor #000000

  BoundaryBorderColor #000000

  ControlBorderColor #000000
  EntityBackgroundColor #FEE2E2
  EntityBorderColor #000000
  LifeLineBorderColor #333333
  LifeLineBackgroundColor #BFDBFE
}

boundary "Trang Quản lý Admin" as boundary

activate boundary

deactivate boundary

admin -> boundary : Tạo sản phẩm mới & Lưu thông tin
activate boundary
boundary -> control : Gửi dữ liệu sản phẩm()
activate control
control -> control : Xác thực thông tin sản phẩm()
control -> entity : Thêm bản ghi sản phẩm mới (stock = 0)()
activate entity
entity -> control : Lưu thành công
deactivate entity
control -> boundary : Thông báo thêm sản phẩm thành công
deactivate control
boundary -> admin : Cập nhật hiển thị sản phẩm mới

activate boundary

activate control
control -> control : Xác thực thông tin đơn nhập kho()

activate entity
entity -> control : Cập nhật CSDL thành công
deactivate entity
control -> boundary : Nhập kho thành công
deactivate control
boundary -> admin : Hiển thị thông báo thành công & Số lượng tồn kho mới
deactivate boundary
@enduml

2.9.6. Lược đồ Tuần tự Xử lý đơn sửa chữa (Kỹ thuật viên)

Kỹ thuật viên sử dụng ứng dụng di động để truy cập chi tiết các đơn sửa chữa được phân công. Hệ thống điều khiển tiếp nhận và truy xuất thông tin thiết bị cùng mô tả lỗi từ cơ sở dữ liệu để phản hồi lên giao diện. Kỹ thuật viên tiến hành kiểm tra máy thực tế, chụp ảnh hiện trạng ban đầu và nhập chẩn đoán cụ thể kèm báo giá dịch vụ. Thông tin này được gửi về hệ thống điều khiển để ghi nhận dữ liệu vào CSDL, đồng thời cập nhật trạng thái đơn sửa chữa sang "Chờ sửa chữa". Nếu có lỗi kết nối CSDL trong quá trình xử lý, hệ thống trả về cảnh báo để KTV thực hiện lại.

⏳ Đang tự động vẽ sơ đồ từ PlantUML...

  EntityBackgroundColor #FEE2E2
  EntityBorderColor #000000
  LifeLineBorderColor #333333
  LifeLineBackgroundColor #BFDBFE
}

control "Hệ thống Xử lý sửa chữa" as control

activate control

activate entity

deactivate entity
control -> boundary : Hiển thị chi tiết lỗi thiết bị
deactivate control
deactivate boundary

tech -> boundary : Nhập thông tin chẩn đoán, giá & chụp ảnh máy
activate boundary
boundary -> control : Gửi thông tin cập nhật chẩn đoán()
activate control

alt #D1FAE5 Hệ thống hoạt động tốt
  control -> boundary : Cập nhật thành công
  boundary -> tech : Hiển thị thông báo trạng thái đơn đã đổi
else #FEE2E2 Gặp lỗi kết nối CSDL / Hệ thống lỗi
  control -> boundary : Báo lỗi hệ thống (Cập nhật thất bại)
  deactivate control

  deactivate boundary

Hình 30. Lược đồ Tuần tự - Xử lý đơn sửa chữa
2.9.7. Lược đồ Tuần tự Báo cáo Thống kê (Admin)

Quản trị viên mở trang báo cáo thống kê trên bảng điều khiển quản trị. Khi Admin lựa chọn mốc thời gian xem báo cáo doanh thu, hệ thống tiếp nhận và gửi truy vấn lọc xuống cơ sở dữ liệu để tổng hợp toàn bộ các đơn hàng và doanh thu linh kiện trong kỳ. Lớp điều khiển xử lý tính toán số liệu thô thu được, chuyển hóa thành biểu đồ trực quan hóa dữ liệu và danh sách sản phẩm bán chạy để phản hồi lên màn hình. Admin cũng có thể gửi yêu cầu xuất file báo cáo. Lớp điều khiển tự động khởi tạo luồng xử lý xuất tài liệu định dạng PDF/Excel từ dữ liệu báo cáo sẵn có và trả về đường dẫn tải xuống cho Admin.

2.9.8. Lược đồ Tuần tự Quản lý Tài khoản & Phân quyền (Admin)

Khi Quản trị viên truy cập trang phân quyền người dùng, hệ thống điều khiển sẽ tự động truy vấn danh sách toàn bộ các tài khoản của khách hàng và nhân viên từ cơ sở dữ liệu để kết xuất hiển thị ra bảng danh sách. Admin lựa chọn một tài khoản cụ thể, thực hiện cập nhật vai trò mới (như nâng cấp lên Kỹ thuật viên) và nhấn lưu. Hệ thống điều khiển tiến hành kiểm định quyền hạn thực thi tối cao của tài khoản Admin hiện tại. Nếu hợp lệ, hệ thống sẽ cập nhật trực tiếp trường vai trò mới của tài khoản tương ứng trong cơ sở dữ liệu và hiển thị thông báo phân quyền thành công.
